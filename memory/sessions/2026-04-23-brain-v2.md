## O que foi feito

Implementamos um **Second Brain** completo para o projeto TPV Sorveteria, baseado em pesquisa de:
- Tiago Forte (PARA + CODE frameworks)
- Niklas Luhmann (Zettelkasten — notas atômicas linkadas)
- Anthropic (context engineering, compaction, structured note-taking)
- JetBrains Research (observation masking vs LLM summarization)
- BMAD Method (agent orchestration com personas especializadas)
- Graham Mann / OpenClaw ("Text > Brain" principle)
- Clawdbot (modular file-based prompts: SOUL + AGENTS + IDENTITY)

## Decisões tomadas (novas ADRs)

- **ADR-008**: CORS no Supabase é CRÍTICO e PENDENTE (configuração manual no dashboard)
- **ADR-009**: Proposta de Edge Function como mitigação de CORS (se dashboard não permitir)

## Lições aprendidas

1. 🧠 CORS é uma restrição do NAVEGADOR. Testar com Node.js/curl NÃO detecta o problema.
2. 🧠 Logs do console do navegador são evidência forense valiosa (encontramos o erro nos logs do Playwright)
3. 🧠 O modo standalone usa localStorage isolado por domínio — não é solução de sincronização
4. 🧠 O brain modular com personas facilita delegação de tarefas: "Act as The Surgeon para debugar"
