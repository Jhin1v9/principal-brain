# AUDIT_RUNBOOK

## Use when

- the goal is exhaustive review
- bugs, regressions or missing tests must be found before changes

## Default team

- CODE_SCOUT
- REVIEWER
- VERIFIER
- SYNTHESIZER

## Mission flow

1. principal defines audit scope and severity bar
2. CODE_SCOUT maps the target system
3. REVIEWER enumerates findings by severity
4. VERIFIER reproduces or refutes critical issues
5. SYNTHESIZER consolidates one audit truth

## Required outputs

- prioritized findings
- evidence for each confirmed issue
- non-reproduced claims
- residual risk map

## Failure conditions

- findings without evidence
- mixing cosmetic feedback with critical bugs
- missing reproduction notes
