# RESEARCH_RUNBOOK

## Use when

- web research is required
- repo mapping is required
- recommendations depend on primary sources and current state

## Default team

- WEB_SCOUT
- CODE_SCOUT
- SYNTHESIZER

## Mission flow

1. principal defines the research question
2. WEB_SCOUT gathers primary external sources
3. CODE_SCOUT maps internal evidence and constraints
4. SYNTHESIZER merges evidence into one state
5. principal writes `STATUS VIGENTE` into `.brain`

## Required outputs

- source list
- evidence list
- internal file map
- ranked risks
- recommended next step

## Failure conditions

- no primary sources
- no internal evidence
- summary without links or file paths
- conflicting conclusions not surfaced
